第一次搭建个人博客

学习了”四大件“，也在leetcode上刷过题，可是在所谓的做项目时却又觉得无从下手。想着要做项目却对要实现什么、怎么实现感到很空泛，又觉得学习过程很漫长。

要不就做个博客吧

我想，记录一下自己总是可以吧，于是就先中断了“怎么去做WebServer”，跑到B站上敲两个字“博客”。

说真的，博客到底是什么，最开始我也不太了解。我记得百科上说是“网络日志”，按照以往看过的博客来说，更像是博主为主体的论坛，只是用户的戏份少甚至没有，这么说的确是“网络日志”，只是成为了Web分享的空间。

感谢 超详细的个人博客搭建教程（无需服务器）- 从原理到实践手把手带你搭建属于自己的个人博客

【雷】Hugo + Github免费搭建博客，并实现自动化部署

我理解了框架是什么（有了感知就不再那么空泛了）、博客怎么去搭建、搭建博客的框架改如何使用。真切地体会到了有意义的行动（虽然没什么难度）。

Hello, Hugo

最后选择了框架Hugo，理由如下：

维度HexoHugo
实现语言Node.js (JavaScript)Go
生成速度慢（文章多会明显变慢）极快（几百篇毫秒级）
安装成本需要 Node 环境单个二进制文件即可
主题数量非常多多，但偏工程风
插件生态npm 插件非常多功能多内置，插件较少
配置风格JS + YAML纯配置 + 模板
模板语言EJS / Pug / Nunjucks 等Go Template（学习成本高点）
稳定性插件多=可能踩坑非常稳定
工程气质前端风后端/系统风

信息来源：chatgpt

说起来是因为Go语言也是编译型语言，所以下载了hugo就能使用了（虽然不是很理解Java为什么非要依赖jvm，尽管我也不会Java）。

完成面向web的个人博客分为四大部分：

  1. 框架构建博客文件
  2. 平台托管文件
  3. 站点部署
  4. CDN加速

ps：Hugo太方便了
